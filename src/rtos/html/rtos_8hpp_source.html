752  // class rtos::task
753  //
755  //
905  //
906  // A task is created by inheriting
907  // from task and providing a main() function. Initialisation of the task,
908  // including creating its waitables, should be done in the constructor.
909  // Don't forget to call the constructor of the task class!
910  //
911  // The main() is the body of the task. It should never terminate.
912  //
913  // Each task has a unique priority (an unsigned integer).
914  // A lower value indicates a higher priority.
915  // The rtos scheduler will always run the task with the higest-priority
916  // runnable (neither blocked nor suspended) task.
917  // A task runs until it changes this 'situation' by using an rtos
918  // call that changes its own state to not runnable, or the state of a
919  // higher priority task to runnable.
920  //
921  // Timers are served only when the rtos is activated
922  // by calling any of its state-changing interfaces.
923  // Hence the longest run time between such calls determines the granularity
924  // (time wise responsiveness) of the application.
925  // Within a time consuming computation a task can call release() to have
926  // the rtos serve the timers.
927  //
